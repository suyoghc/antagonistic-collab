Arbiter Comparison: George's Original → Our Categorization Implementation

Component George's Original (ARBITER/Crucible) Our Categorization Implementation
Framing & Goals
Core Question
  • How should you structure LLM-mediated scientific debate so it's productive?
  • Builds a tool for scientists to use
    • Orchestration framework for resolving disputes
    • Assumes debate is valuable; designs the best way to conduct it
  • Does LLM debate actually help beyond computation alone — and if so, when?
  • Treats George's framework as a hypothesis to test rather than a tool to deploy
  • Testing this required specific infrastructure:
    • Synthetic data → known ground truth
    • Bayesian scoring → automated measurement
    • Factorial ablation → causal isolation
  • These requirements drive most architectural differences below
Δ: George asks "how should debate work?" — we ask "does debate work?" Testing the hypothesis required infrastructure (ground truth, measurement, ablation) that a deployment tool doesn't need.
Key Takeaway
  • Debate is valuable; the question is how to structure it well
  • Debate is conditionally valuable — epiphenomenal under correct specification, causally necessary under misspecification
  • Every component carries an implicit prior: computation is neutral, debate favors interpretability, the arbiter favors complexity
  • More sophisticated LLM coordination makes things worse — the arbiter degrades performance in both domains
Δ: George assumes debate is generally beneficial. We show it's conditionally beneficial — and that each layer of LLM involvement introduces a systematic bias.
Implications
  • For hybrid AI systems: LLM-computation orchestration is a viable paradigm for automated science
  • For model selection: OED can be augmented with natural-language reasoning about theories
  • For philosophy of science: Adversarial collaboration can be structured and automated
  • For hybrid AI systems: Each LLM layer adds an implicit prior — characterize your biases before deploying. The optimal architecture is not "all layers active" but depends on specification quality. Less LLM coordination can outperform more.
  • For model selection / OED: Standard EIG fails under misspecification because it optimizes for the wrong parameter regime. LLM semantic diagnosis provides the parameter estimation that OED lacks — but only for parameters with linguistically describable effects. A new form of experimenter bias: the selection method itself biases which model wins.
  • For philosophy of science: Automated adversarial collaboration reproduces Lakatos (agents protect core theories by modifying auxiliary assumptions ~85% of the time, despite having no career incentives — suggesting the "protective belt" is structural, not psychological), reveals that "impartial" experiment selection carries implicit epistemic values, and shows that more structured methodology can degrade convergence — the coordination paradox.
Practical Aim
  • Resolve scientific disputes between competing theories
  • Produce structured report:
    • Areas of agreement
    • Areas of disagreement
    • Next steps
  • General-purpose adversarial collaboration framework
  • Determine which cognitive model best explains categorization behavior
  • Quantify model fit via Bayesian posterior
  • Characterize when debate is causally necessary vs. epiphenomenal
Δ: Narrowed from general dispute resolution to specific model comparison with quantitative outcome. The practical aim is subordinate to the core question above.
Scientific Domain
  • Domain-agnostic — designed for any scientific controversy
  • Example instantiation: developmental psychology (nativist vs empiricist)
  • Category learning (Shepard et al., 1961 structures)
    • 6 category types (Type I–VI), varying difficulty
    • Rich computational tradition (Nosofsky, Love, Nosofsky & Palmeri)
Δ: Instantiated to a specific domain with known ground truth and established computational models.
Models Under Comparison
  • Any competing theories — specified at session setup
  • Example: 2 theories with qualitative predictions
  • Framework supports both qualitative and quantitative theories
  • Three computational models, all with quantitative predictions:
    • GCM (exemplar similarity) — 3 params
    • SUSTAIN (cluster-based) — 4 params
    • RULEX (rule-plus-exception) — 2 params
  • Three representational formats: similarity, clusters, rules
Δ: Fixed to 3 specific computational models, all with quantitative predictions. George's framework also supports qualitative theories; ours requires numerical output for Bayesian scoring.
Theory of Debate Value
  • Adversarial collaboration surfaces hidden assumptions
  • Structured disagreement is more productive than unstructured
  • Human moderator ensures fairness
  • Debate enables parameter recovery that EIG alone cannot do
    • LLMs diagnose why predictions are wrong, not just that they're wrong
    • Complementary biases: EIG finds informative experiments, debate fixes parameters
  • Debate is epiphenomenal under correct specification (M13–M14)
Δ: George's framework posits debate as generally beneficial. Our findings narrow this to a specific causal claim: debate matters for parameter recovery under misspecification, and is epiphenomenal otherwise.
Architecture & Pipeline
Overall Architecture
  • CrucibleSession orchestrates 6 sequential phases
  • Moderator agent controls flow
  • Fully LLM-driven with HITL checkpoints
  • run_debate_cycle() — EIG computation + LLM debate rounds
  • Crux negotiation replaces free-form debate
  • Arbiter is optional layer on top
  • Deterministic orchestration replaces Moderator agent
Δ: Replaced free-form phases with structured crux protocol. Computation (EIG) drives experiment selection. HITL checkpoints removed — fully automated.
Agent Roles
  • 4 agent types:
    • Theory Agent — one per model, advocates for its theory
    • Moderator — orchestrates debate, ensures fairness
    • Integrator — finds common ground, proposes syntheses
    • Critic — stress-tests claims, identifies weaknesses
  • 3 agent types:
    • Model Agent — one per model, with computational model access
    • Integrator — meta-agent, synthesizes crux map (arbiter only)
    • Critic — meta-agent, challenges leading model (arbiter only)
  • Moderator role replaced by deterministic orchestration code
Δ: Moderator → code. Theory agents enhanced with computational model access. Integrator/Critic scoped to arbiter layer only, rather than active across all phases.
Number of Phases / Cycle Structure
  • 6 sequential phases:
    • Setup → Structured Debate → Integration → Synthesis → Report → Research Program
  • Single pass
  • 1 cycle =
    • EIG select → data gen → posterior update → debate round (→ optional arbiter)
  • Repeat for N cycles (typically 5)
  • Iterative rather than single-pass
Δ: Collapsed 6 sequential phases into a tight iterative compute-debate loop. Report and research-program phases handled outside the system.
Debate Format
  • Free-form natural language debate
    • Moderator poses questions, agents respond
    • Unstructured argumentation across multiple rounds
  • Structured crux negotiation: propose → accept/reject/counter
  • Each round provides:
    • Prediction errors per model on each structure
    • Prompt for parameter revisions
  • JSON-formatted responses parsed into:
    • Crux proposals
    • Parameter change proposals
Δ: Free-form debate → structured protocol. Agents make specific, testable claims. Concrete per-model prediction data rather than narrative argument.
Experimental Conditions
  • Single mode: full pipeline runs end-to-end
  • All components always active
  • Three conditions tested as independent variables:
    • No-debate: computation only (EIG + posterior)
    • Debate: adds crux negotiation + parameter recovery
    • Debate + arbiter: adds Integrator/Critic meta-agents
  • Factorial design across M15–M17
    • Also crossed with misspecification (correct/wrong params)
    • Also crossed with design space (open/closed)
Δ: George's pipeline is monolithic. We decomposed it into independently toggleable components, enabling ablation and causal attribution via factorial design.
Key Mechanisms
Experiment Selection
  • LLM-proposed via Moderator
  • No computational optimization
  • Experiments emerge from debate
  • EIG (Bayesian Expected Information Gain) — primary
    • Computationally optimized for model discrimination
    • Thompson sampling to balance exploration/exploitation
  • Crux-directed mixture (30% crux weight)
  • Open design: LLM proposals as supplement (M16)
Δ: Computational optimization (EIG) replaces LLM-proposed experiments. LLM proposals retained as an optional supplement in open-design mode.
Crux Mechanism
  • No formal crux system
  • Disagreements emerge organically in Phases 2–3
  • Formal crux protocol: propose → accept/reject/counter
  • Each crux has:
    • Type (prediction, parameter, mechanism)
    • Target structure
    • Confidence level
  • Auto-resolve against new experimental data
  • Crux map tracks all active disagreements
Δ: Entirely new mechanism with no counterpart in George's design. Formalizes disagreement into typed, testable claims that resolve automatically against experimental data.
Parameter Recovery
  • Not addressed — framework focuses on model comparison rather than parameter estimation
  • Assumes models are correctly specified throughout
  • Pipeline: sync_params_from_theory() + validate_param_revision()
    • Agents propose param changes during debate
    • Gated by per-cycle RMSE improvement
  • Results: 60–86% recovery under misspecification (M15)
Δ: Entirely new mechanism. George's framework assumes correct specification; ours adds parameter recovery as a debate capability. This turns out to be what makes debate causally necessary.
Meta-Agents (Integrator / Critic)
  • Integrator:
    • Finds common ground
    • Proposes syntheses
  • Critic:
    • Stress-tests claims
    • Identifies weaknesses
  • Both active across all phases
  • Operate in natural language
  • Integrator:
    • Synthesizes crux map → experiment suggestions
  • Critic:
    • Challenges leader, proposes falsification experiments
  • Active only in arbiter condition (vs. across all phases in George's)
  • Output feeds into crux-directed experiment selection (vs. open-ended debate participation)
  • Key finding: meta-agents are biased
    • Help similarity models (SUSTAIN +8pp)
    • Hurt rule models (RULEX −22pp)
Δ: Scoped from free-form participants to crux-focused advisors. Empirically discovered they introduce a model-type bias rather than uniform improvement.
Conflict Resolution
  • Moderator-mediated dialogue
    • Integrator proposes resolutions
    • Critic challenges proposals
  • Human makes final call at checkpoints
  • Crux auto-resolution against experimental data
    • New data addresses crux → auto-confirm or auto-refute
    • Cruxes are testable empirical claims, designed to be resolvable by data
  • No human in loop
Δ: Data-driven auto-resolution replaces human-mediated negotiation. George's human checkpoints ensure fairness; our data gates ensure empirical grounding.
Open Design Space
  • All experiments are LLM-proposed
  • Design space is always "open" (unconstrained)
  • Two modes:
    • Closed: registry only, EIG selects
    • Open: agents can propose novel structures beyond registry
  • M16 tested both as controlled experimental conditions
  • Open helps RULEX (+24pp), hurts SUSTAIN (−24pp)
Δ: Added a registry-bounded mode alongside open design. George's framework is always open; ours treats openness as a manipulable experimental variable.
Infrastructure & Methodology
Data Source
  • Designed for real experimental data
  • Theories evaluated against empirical literature
  • Synthetic data from ground-truth model
    • n=30 simulated subjects per structure
    • Known correct answer enables automated validation at scale
  • Real human data untested (listed as future work)
Δ: George's framework is designed for real data; ours uses synthetic data to enable automated validation. Trades ecological validity for experimental control and scalability.
LLM Backbone
  • OpenAI API (GPT-4 assumed)
  • Each agent is a separate system prompt + conversation
  • LLM handles reasoning, scoring, and experiment design
  • GPT-4o via OpenAI API
  • LLM does:
    • Interpretation and diagnosis
    • Parameter proposals
    • Crux negotiation
  • LLM does NOT do:
    • Scoring
    • Experiment selection
    • Predictions — these are all computational
  • Single backbone — multi-LLM untested
Δ: Strict division of labor — LLM for semantics, computation for numerics. George's LLM handles the full pipeline; ours restricts LLM to tasks where it has comparative advantage. Largest single architectural departure.
Scoring / Model Comparison
  • Qualitative evaluation
    • Integrator synthesizes arguments
    • Panel evaluates "consilience" and "predictive accuracy" narratively
  • No formal metric
  • Bayesian posterior over models
    • Learning-curve RMSE + LOO cross-validation
    • Winner = highest posterior after N cycles
    • Gap metric = margin between winner and runner-up
Δ: Quantitative Bayesian scoring replaces narrative evaluation. George's approach captures nuances that numbers miss; ours enables automated, reproducible comparison.
HITL Checkpoints
  • 6 checkpoints:
    • Initial Review
    • Evidence Evaluation
    • Integration Review
    • Final Synthesis Review
    • Report Approval
    • Research Program Approval
  • None — fully automated
  • Human reviews results post-hoc
Δ: All 6 checkpoints removed to enable automated experimentation at scale. George's checkpoints serve quality control and fairness; ours trades that for throughput and reproducibility.
Convergence / Stopping Rule
  • Moderator decides when to advance phases
  • Human approves at checkpoints
  • No formal convergence criterion
  • Fixed N cycles (typically 5)
    • No early stopping — run all cycles
    • Posterior convergence observable but not used as stopping rule
Δ: Fixed cycle budget replaces adaptive human stopping. Simpler and more reproducible, though may waste cycles or stop too early.
Validation Method
  • Human expert evaluation of final report
  • No automated validation
  • Success = "useful report produced"
  • Automated metrics:
    • Correct winner identified? (binary)
    • Gap: winner's posterior margin
    • Parameter recovery %: closeness to ground truth
  • Ablation: debate vs no-debate vs arbiter
  • 47/48 correct across M14–M17
Δ: Automated validation with known ground truth, plus ablation design to isolate causal contribution of each component. George's expert evaluation captures richer judgments; ours scales.
Results & Findings
Key Results
  • Framework and codebase published
  • Correct specification (M13–M14):
    • 18/18 correct — debate adds no value
  • Misspecification (M15):
    • 8/9 correct — debate causally helps
    • +3.5pp GCM, +22.4pp RULEX
  • Arbiter (M16):
    • 15/15 correct — arbiter is biased (helps SUSTAIN, hurts RULEX)
  • Composition (M17):
    • 6/6 correct — non-additive; open design rescues RULEX from arbiter catastrophe
  • R-IDeA: negative result — diversification antagonistic to debate
  • Overall: 47/48 correct
Δ: Ran extensive empirical validation (47+ runs). Key finding: debate is conditionally valuable — specifically under misspecification — rather than universally beneficial.
Known Failure Modes
  • Anticipated risks:
    • LLM confabulation
    • Unfair moderation
  • Empirical characterization pending
  • Arbiter-related:
    • Arbiter + RULEX: meta-agents distort divergence mapping → wrong winner (3.2%, M15)
  • Parameter-related:
    • SUSTAIN misspec: invisible to debate — not enough prediction error (0% recovery)
    • Abstract params: LLMs can't diagnose without linguistic interpretation (SUSTAIN eta, r)
  • Selection-related:
    • Greedy EIG: picks same structure repeatedly → Thompson sampling fixes
  • LLM type instability: params arrive as wrong types (scalar vs list, dict vs string)
Δ: Empirically discovered through 47+ runs. Each failure mode led to a specific architectural fix or documented limitation.
